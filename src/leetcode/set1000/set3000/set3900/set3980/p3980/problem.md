# 3980. 变换二进制字符串的最少操作次数

[Problem link](https://leetcode.cn/problems/minimum-operations-to-transform-binary-string/description/)

[Reference solution](https://leetcode.cn/problems/minimum-operations-to-transform-binary-string/solutions/3991753/zhuang-tai-ji-dp-by-yinsen-o-76gf/)

给你两个长度同为 `n` 的二进制字符串 `s1` 和 `s2`。

可以对 `s1` 执行任意次下面两种操作：

- 选择一个值为 `'0'` 的位置，把它改成 `'1'`。
- 选择一对相邻位置 `i` 和 `i + 1`，要求这两个位置当前都为 `'1'`，然后把它们都改成 `'0'`。

返回把 `s1` 变成 `s2` 的最少操作次数；如果无法做到，返回 `-1`。

## Function signature

```go
func minOperations(s1 string, s2 string) int
```

## Examples

### Example 1

```text
Input: s1 = "11", s2 = "00"
Output: 1
```

把相邻的两个 `'1'` 一次变成 `'0'` 即可。

### Example 2

```text
Input: s1 = "01", s2 = "10"
Output: 3
```

一种最优做法是先把第一个字符从 `'0'` 改成 `'1'`，得到 `"11"`；再把相邻的两个 `'1'` 改成 `"00"`；最后把第一个字符改成 `'1'`，得到 `"10"`。

### Example 3

```text
Input: s1 = "1", s2 = "0"
Output: -1
```

单个 `'1'` 无法通过相邻二元操作变成 `'0'`。

## Constraints

- `1 <= n == s1.length == s2.length <= 100000`
- `s1` 和 `s2` 仅由 `'0'` 和 `'1'` 组成。

## Solution Summary

从左到右处理字符串。处理到位置 `i` 时，之前的位置已经固定成 `s2`，后续操作不能再影响它们。当前位置只需要知道一个状态：它当前是 `0` 还是 `1`。

令 `dp[0]` 和 `dp[1]` 分别表示处理到当前位置 `i` 前，且当前位置当前值为 `0` 或 `1` 的最小代价。当前位置的值可能来自原始 `s1[i]`，也可能是上一步对 `(i-1, i)` 执行相邻二元操作后被变成了 `0`。

对每个位置有两类选择：

- 不使用 `(i, i+1)` 的二元操作。此时必须直接把当前位置固定成 `s2[i]`。如果目标是 `1` 且当前是 `0`，可以花一次操作把它改成 `1`；如果目标是 `0` 且当前是 `1`，这种选择不可行。下一位保持它在 `s1` 中的原始值。
- 使用 `(i, i+1)` 的二元操作。先把当前位置和下一位置都变成 `1`，需要的代价是当前为 `0` 时加一、下一位原始值为 `0` 时加一；然后花一次操作把二者都变成 `0`。如果 `s2[i]` 是 `1`，再额外花一次操作把当前位置改回 `1`。下一位状态变成 `0`。

最后一个位置没有右邻居，所以只能用第一类选择来收尾。如果最后无法得到目标值，答案为 `-1`。

复杂度：

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(1)`。

### Why this is simpler than the old solution

原来的递归写法也在表达同一个状态机，但它把“下一位提前从 `0` 改成 `1`”也作为状态转移的一部分，因此分支比较绕。实际上，如果没有对 `(i, i+1)` 执行二元操作，提前改下一位并不会影响当前位置，完全可以等处理下一位时再决定是否把它从 `0` 改成 `1`。

因此迭代版只保留两种真正必要的状态：当前位置当前是 `0` 还是 `1`。这样既避免了递归深度问题，也让每一步的两种选择更直接。
